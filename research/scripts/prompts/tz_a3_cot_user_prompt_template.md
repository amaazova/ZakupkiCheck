Проанализируй следующее техническое задание (ТЗ) госзакупки по 44-ФЗ и извлеки все потенциальные проблемы соответствия согласно описанной в system prompt схеме.

{extra_instruction}

Проанализируй этот документ **пошагово**:

1. Сначала определи тип объекта закупки (товар, работа, услуга) и его область — медицина / IT / строительство / транспорт / прочее.
2. Проверь наличие торговых марок, конкретных моделей или производителей. Для каждого упоминания зафиксируй, есть ли рядом оговорка «или эквивалент» / «или аналог».
3. Оцени полноту описания характеристик: указаны ли единицы измерения, диапазоны значений (от/до, не менее, не более), функциональные и технические требования.
4. Найди ограничительные формулировки, которые могут сужать круг потенциальных поставщиков: «единственный», «только», «исключительно», узкие диапазоны под одного производителя, субъективные критерии.
5. Проверь соответствие КТРУ / ОКПД2: указан ли код, есть ли характеристики сверх позиции каталога, обоснованы ли они.
6. Проверь сроки и порядок приёмки/экспертизы: указаны ли критерии приёмки результата, нет ли противоречивых сроков.
7. На основе анализа сформируй итоговый JSON по схеме из system prompt.

Не выводи промежуточные рассуждения — только финальный JSON. Внутренний CoT — это руководство к тому, ЧТО проверить, а не к тому, ЧТО возвращать. Ответ должен содержать ТОЛЬКО валидный JSON без пояснений и без markdown-фенсов.
