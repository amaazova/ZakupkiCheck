# `baselines/` — baseline-эксперименты

Эта папка содержит код **предварительных экспериментов** из Введения и
Приложения ВКР, на фоне которых обоснована двухпроходная архитектура и
выбор Knowledge Ladder.

## Что внутри

### `eval_pipeline_v9/` — v9 detector pipeline (B4)

Пайплайн из четырёх независимых LLM-детекторов: `brand_check`,
`completeness_check`, `ktru_check`, `restriction_check`. Каждый детектор —
отдельный LLM-вызов с собственным узким промптом, а агрегатор склеивает
результаты. **Проблема, которую выявил этот эксперимент:** детекторы не
видят друг друга и плодят дубли (бренд с оговоркой «или эквивалент»
правильно не флагуется `brand_check`-ом, но `restriction_check` отдельно
флагует ту же марку, не зная про оговорку — результат: 9 finding'ов там, где
реальных проблем 1–2). Именно это поведение мотивировало переход к
двухпроходной архитектуре в `app/`.

### `streamlit_v1/` — однопроходный Streamlit-прототип (§3.3.1)

Самая первая версия сервиса: один LLM-вызов на ТЗ с открытой схемой
(`tz_l1_*` промпты), вывод JSON напрямую с типами нарушений и
доверительными значениями. Используется в ВКР как baseline для сравнения с
двухпроходной v2.

### `eval_v9_scripts/` — baseline-эваляции из Введения ВКР

- `eval_b0_majority.py` — majority-class baseline (Введение, §4.1).
- `eval_b1_regex.py` — regex-only baseline (Введение).
- `eval_b2_zeroshot.py` — zero-shot LLM без taxonomy hint (Введение).
- `eval_b3_taxonomy.py` — LLM + taxonomy hint, без few-shot (Введение).
- `eval_b4_pipeline.py` — четырёх-детекторный baseline (Введение, §4.1; использует `eval_pipeline_v9/`).
- `eval_memorization_probe.py` / `eval_memorization_run.py` — тест на
  memorization (50 эпизодов × shifted-вариант, §4.2). Импортирует
  `eval_pipeline_v9.detectors._common` — поэтому удаление `eval_pipeline_v9/`
  сломает memorization probe.

## Зачем сохранено

ВКР опирается на эти результаты, поэтому полностью убирать код нельзя — без
него Введение и Приложение становятся невоспроизводимыми. Но и в основной
линии сервиса они не нужны: актуальная архитектура — двухпроходная
(`app/`).
